# Contributing to bookmetrics

Thank. Thank you for your interest in contributing to `bookmetrics`! This package is designed to bring quantitative narrative analysis to public-domain literature, and we welcome improvements, new features, and bug fixes.

## Project Architecture

`bookmetrics` follows a modular structure:

- **Core Logic**: Located in the `R/` directory.
    - `analyse_book.R`: The primary entry point for book processing.
    - `registry.R`: Internal mapping of identifiers to Gutenberg URLs.
    - `load_gutenberg_book.R`, `split_into_chapters.R`, etc.: Low-level text processing and analysis utilities.
- **Visualizations**: Found in the `R/` directory (e.g., `plot_*.R`). These use `ggplot2`.
- **Testing**: Located in `tests/testthat/`. Every function should have corresponding tests.
- **Documentation**: 
    - `README.md`: The primary landing page and user guide.
    - `man/`: Package documentation (generated by Roxygen2).
    - `pkgdown/`: Built website for browsing documentation.

## How to Add New Features

### 1. Adding a new analysis step (e.g., "Theme detection")
- Implement the logic in a new file `R/theme_detection.R` or within an existing one.
- Update `analyse_book()` in `R/analyse_book.R` to include this new component in the pipeline.
- Add corresponding tests in `tests/testthat/`.

### 2. Adding a new plot (e.g., "Character network")
- Create a function in `R/plot_network.R` that takes an `book_analysis` object and returns a `ggplot2` object.
- Ensure the function uses `theme_bookmetrics()` for consistency.
- Update `generate_gallery()` in `R/generate_gallery.produ_r` to include this new plot in the gallery generation loop.
- Add an integration test in `tests/testthat/test-e2e_integration.R` ensuring it is included in the HTML output.

### 3. Adding a new narrative skill (e.s., "Character Proximity")
- Implement the computation logic in `R/`.
- Integrate it into the pipeline in `R/analyse_book.R`.
- Ensure the resulting data is stored within the `book_analysis` object.

## Workflow

### Testing Workflow
Before submitting any changes, ensure all tests pass:
```r
library(testthat)
testdir("tests/testthat")
```
Always run the end-to-end integration test to ensure new features don't break the gallery or analysis pipeline.

### Documentation Workflow
- **Functions**: Always use Roxygen2 comments (`#'`) for all exported functions.
- **README**: Update `README.md` if you introduce major new capabilities or change usage patterns.
- **pkgdown**: After updating documentation, rebuild the site:
```r
library(pkgdown)
build_site()
```
